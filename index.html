Project-long Course Project as part of COSI 116A: Information Visualization, taught by Prof. Dylan Cashman, Brandeis University.

Motivation

This project aims to analyze changes in the software market by examining trends in the most popular programming languages. We included comparisons of different languages tutorial search rate during times. Performance comparisons across various algorithms and tasks, focusing on runtime and memory usage. In addition, we compared usage rates across different regions, aiming to uncover how cultural, educational, or economic factors might influence language preferences. Through these visualizations, we hope to help newcomers make more informed decisions when selecting the “best” or “most popular” programming language as a starting point, while also offering experienced professionals insights into broader market trends and future directions.

The rapid evolution of programming languages presents challenges for developers, educators, and organizations in making informed decisions about which languages to learn, teach, or adopt. While some languages gain widespread popularity due to advancements in frameworks or industry adoption, others decline or remain niche. There are many different programming languages, each with its own syntax, features, and typical use cases. Some languages may excel in system-level programming, while others thrive in data analysis, artificial intelligence, or web development. When a particular language is widely adopted in a certain field, it often reflects the direction of research, the pace of technological advancement, or even the strength of related job markets. By understanding these dynamics, both aspiring and seasoned developers can better align their skill sets with emerging opportunities and evolving industry demands.

Visualization explanation

Our project focuses on creating an interactive data visualisation system that integrates a table, map, line chart, and scatter plot to provide insights into the popularity of different programming languages. The Barchat at the top show the trend on the popularity languages by Analysis the times a language tutorial is serached. User can move the slider below the barchart to see the data at a specific time. Since the focus here is on analyzing trends, a bar chart was used because its advantages on clear visual comparison and it also shows the growth or decline of a language from 2004 .

You can select a specific time by dragging the slider below the bar chart, which will update the chart to display data from that particular point in time. You can also click the play button to have the bar chart advance automatically month by month from the current position, and pause at any time to make adjustments.

When you brush on the bar chart, the table below will highlight the selected languages. The table is sorted based on data from 2004, allowing you to compare the current popularity of these languages with their 2004 levels of popularity.

The World map displays countries colored based on data availability. When a user clicks on a country, the bar chart below updates to show the percentage of developers in that region using different programming languages. When hovering over a bar, the map dynamically displays a heatmap that represents the popularity of the corresponding programming language across different regions. The interactive nature of the map and charts allows users to explore the data for each region, offering insights into global programming language trends.

The third visualization integrates a Scatterplot and a table to provide an interactive exploration of programming language trends. The scatterplot visualizes the relationship between runtime and memory usage, using color-coded points to represent different languages. The table complements this by listing detailed information, such as language type and average CPU load, which users can sort dynamically. Through brushing and linking, users can select data points on the scatterplot to highlight corresponding rows in the table, and vice versa, enabling synchronized and intuitive data exploration.

    • Data Analysis

    Summary of data, data types, and data preprocessing.

    The data used in the barchart and slider is in the programming_language_trends.csv

    It include 4 types of data: date,category,value,color. each language has a specific color assigned to, and each language has corresponding data for the 12 months of each year. After organizing them, the data types are read to generate the visualization.

    The data for world map: programming_languages_popularity_by_region.csv.

    It includes data on regions (category) and the percentage values for each programming language in those regions. The organized data is then read to generate the visualization.

    The data for Scatterplot and Table: Scatterplot_data.json.

    The dataset consists of programming language performance metrics, including runtime, memory usage, source language, and average CPU load. Each record corresponds to a specific data point representing a programming language's performance characteristics.

    Task Analysis

    Task1: Identify the most popular programming languages based on recent trends.

    Task1: The bar chart clearly reflects the differences in popularity among different programming languages, as the bars are placed side by side, making it easy to compare their lengths. Additionally, the chart is already sorted, making the overall trend even clearer. Bar charts are also well-suited for handling categorical data, and this chart allows us to observe and analyze gradual trend changes over time by moving the time bar below the chart. By using encoding on categorized languages and date, the chart allows viewers to compare values effectively and complete the task1 well. And the selection feature of the table below allows viewers to better choose specific data to display, making it easier to compare and highlight the differences in data size for a smaller number of languages.

    Task2: Compare performance of programming languages for different tasks.

    Task2:The scatterplot visually represents relationships between runtime (x-axis) and memory usage (y-axis), with point sizes encoding average CPU load. The clear separation of points and their color-coded representation by language makes it intuitive to compare multiple languages at a glance. By selecting regions of interest using the brushing feature, users can isolate high-performing or outlier languages for specific tasks. The table complements the scatterplot by displaying detailed data, such as language type and other performance metrics, in a sortable format. Sorting and filtering features enable users to refine their analysis, focusing on relevant languages or metrics.

    The linking functionality between the scatterplot and table ensures synchronized updates—brushing a region in the scatterplot highlights corresponding rows in the table, and vice versa. This seamless interaction allows users to analyze specific subsets of languages, compare their performance, and identify the best options for different computational needs. The combination of scatterplot encoding and table interaction provides a powerful tool to evaluate trade-offs in runtime, memory usage, and CPU load, effectively addressing the task of comparing programming language performance across various scenarios.

    Task3: Programming Languages Popularity by Region.

    Task3: The processes involved in achieving the goal include browsing the map to select a country, then exploring the accompanying charts that dynamically update based on user selection. Additionally, hovering over a bar in the chart triggers a heatmap on the map, visually representing the popularity of the selected programming language across different regions. The abstract tasks supported by this visualization include comparison of language popularity across regions, exploration of trends within a specific country, discovery of regional differences that may not be immediately evident without the use of interactive visual tools, and analysis of language-specific global distributions.

    Design Process

    Sketches and design choices to justify final visualization.

    The three images above are the initial three designs we decided to use

    The two images above show the earliest designs for the bar chart. However, due to time constraints and issues with data linkage at a later stage, we abandoned the feature of selecting and displaying only certain languages.

    The two images above shows the designs of scatter plot. In the initial design phase, I envisioned incorporating textual descriptions to highlight the unique features of each programming language directly within the visualization. However, due to time constraints and implementation complexity, I decided to prioritize enhancing the interactive and filtering capabilities of the charts. By focusing on features like brushing, dynamic linking, and sorting, users can now explore the data in a more intuitive and flexible way. This approach not only improves the usability of the scatterplot and table but also allows users to analyze performance metrics such as runtime, memory usage, and CPU load for specific programming languages in greater detail.

    During implementation, we discovered that the data we had didn't fit well with our initial pie chart design, so we opted for the second design, the map.

    Conclusion

    Summary of Work Completed

    This project successfully developed an interactive data visualization system to explore the trends and performance of programming languages. The system integrates multiple visualization techniques:

    • Bar Chart with Time Slider: Highlights trends in the popularity of programming languages over time, allowing users to analyze changes and make direct comparisons.
    • World Map and Heatmap: Provides insights into regional variations in language usage and popularity, enabling geographic-specific analysis.
    • Scatterplot and Table: Facilitates detailed performance comparisons across languages, showing runtime, memory usage, and CPU load, while allowing users to interactively select and sort data for specific tasks.

    The brushing and linking functionality across visualizations ensures a seamless exploration experience, enabling users to connect high-level trends with detailed performance metrics effectively.

    Areas for Improvement

    • Data Completeness: Some regions or programming languages have limited or no data. Enhancing the dataset with more comprehensive information would improve analysis accuracy.
    • Scalability: For larger datasets, the visualizations may become crowded or unresponsive. Optimizing the visual components for performance and incorporating zoom or filtering features could address this issue.
    • User Interface: The current UI could be enhanced to provide more intuitive controls, such as tooltips for scatterplot points and map regions, or a search feature for quick access to specific languages or regions.
    • Mobile Responsiveness: The visualizations are optimized for desktop devices but could be adapted for mobile screens to reach a wider audience.

    Future Work

    • Advanced Trend Analysis: Incorporate machine learning models or statistical analysis to predict future trends in programming language popularity based on historical data.
    • Additional Metrics: Extend the scatterplot and table to include new metrics, such as developer satisfaction, learning difficulty, or language-specific tooling support.
    • Dynamic Data Integration: Allow users to upload their own datasets to analyze trends specific to their contexts, further enhancing the system's versatility.
    • Improved Visual Encoding: Explore alternative visualizations (e.g., radar charts for multiple metrics or treemaps for hierarchical data) to provide a broader perspective on language performance and popularity.
